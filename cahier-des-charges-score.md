# Cahier des Charges - Site Web SCORE

## 1. Identite

Nom : SCORE

Signification : Software Control Optimization Risk & Evaluation

Activite principale : Entreprise de Services Numeriques (ESN) specialisee dans :
- Les tests d'applications web et mobiles
- L'assurance qualite logicielle (QA)
- L'evaluation des risques lies aux applications
- Le controle qualite des solutions digitales

Activites complementaires :
- Developpement web
- Digitalisation des processus metiers
- Support technique informatique

## 2. Positionnement Et Objectif Principal

Positionnement : SCORE aide les entreprises a detecter les anomalies, dysfonctionnements et risques presents dans leurs applications avant qu'ils n'impactent leurs utilisateurs.

Objectif principal du site : Inciter les visiteurs a soumettre leur application afin d'obtenir une evaluation gratuite et decouvrir les services d'assurance qualite proposes par SCORE.

## 3. Message Principal De La Page D'accueil

Titre principal : Faites tester votre application gratuitement sous 72 heures ouvrees.

Sous-titre : Recevez un premier rapport d'evaluation de votre application web ou mobile par notre equipe QA.

## 4. Menu Du Site

Accueil : Presentation rapide de SCORE et formulaire de soumission.

Nos services :
- Tests logiciels : Tests fonctionnels, tests exploratoires, tests de regression, tests de validation.
- Assurance qualite : Verification de conformite, audit qualite, revue fonctionnelle, accompagnement QA.
- Developpement web : Sites vitrines, applications web, solutions metier.
- Digitalisation : Automatisation de processus, outils de gestion, solutions numeriques.
- Support technique : Assistance utilisateurs, maintenance applicative, support informatique.

## 5. Nos Offres

Offre Decouverte (Gratuite) :
- Analyse preliminaire
- Identification des anomalies visibles
- Rapport synthetique
- Retour sous 72 heures ouvrees

Offre Standard (49 000 FCFA) :
- Analyse fonctionnelle complete
- Cas de tests
- Rapport detaille
- Recommandations
- Support apres restitution

Offres avancees : Details a definir selon les besoins specifiques des clients.

## 6. Pourquoi Tester Son Application ?

- Reduire les risques de bugs en production
- Ameliorer l'experience utilisateur
- Renforcer la confiance des clients
- Identifier les anomalies avant lancement
- Reduire les couts de correction

## 8. Formulaires Et Soumissions

### 8.1. Formulaire De Soumission D'application

Informations sur l'entreprise :
- Nom de l'entreprise
- Nom du responsable
- Email
- Telephone

Informations sur l'application :
- Nom de l'application
- Type d'application : Site web, application mobile Android, application mobile iOS, application metier, autre
- URL de l'application
- Identifiants de test, facultatif
- Description du besoin
- Commentaires supplementaires

Validation :
- Case a cocher obligatoire : "J'autorise SCORE a acceder a mon application uniquement dans le cadre de son evaluation."

Actions post-soumission :
- Envoi automatique d'un email recapitulatif a l'administrateur : obtenezinfoscore@gmail.com
- Affichage et/ou envoi d'un message de confirmation a l'utilisateur : "Votre demande a bien ete enregistree. Notre equipe analysera votre application et vous transmettra un retour sous 72 heures ouvrees."

### 8.2. Formulaire De Contact Classique

- Message
